INTRODUCTION TO PROGRAMMING

Lesson 1: Fundamentals of the Web and HTML

Basic Concepts

The World Wide Web (or the Web):The World Wide Web (www) is an information system of interlinked hypertext documents-it's sort of a collection of HTML documents-that are accessed via the Internet. It has also commonly become known simply as the Web. Each file on the web can be located through its uniform resource locator (or URL). For more on the world wide web see [Wikipedia].

The Internet: The Internet is a global system of interconnected computer networks that use the standard Internet protocol suite (TCP/IP) to link several billion devices worldwide.

Web Pages: Individual document pages on the World Wide Web are called web pages-there are over 30 billion pages. Web pages may contain text, images, videos, and other multimedia components, as well as web navigation features consisting of hyperlinks.

Web Browser: Web pages are accessed with a software application running on the user's computer, commonly called a web browser.

Hypertext Markup Language (HTML): HyperText Markup Language, commonly referred to as HTML, is the standard markup language used to create web pages. It is written in the form of HTML elements consisting of tags enclosed in angle brackets. HTML is made up of text content, markups-made up of things called tags, references to other documents, images, videos and links to other pages. HTML documents contain text content which describes "what you see" and markup which describes "how it looks" when, for instance, you visit a web page. For a more thorough explanation click here .

Html Tags and Elements: HTML documents are made of HTML elements. An HTML element refers to everything within a set of opening and closing tags.

When writing HTML, we tell browsers the type of each element by using HTML tags. An explanation of how this works can be found by clicking here. Examples of tags are: "b" for bold, "i" for italics, "em" for emphasis-italics, "p" for paragraph, "a" for anchor, "br" for break or void tag, "img" for images.

Tags can have one or more attributes. An attribute is a property of an HTML element.

Inline vs Block Elements: HTML elements are either inline or block. While Inline elements look linear, Block elements form an "invisible box" around the content inside of them.

Thinking Like a Programmer

To think like a programmer you must develop procedural thinking, abstract thinking, systematic debugging (the skill through which programmers identify problems and/or potential problems and then track down potential causes for those problems until they are found and fixed), technological empathy (understanding how a computer see things), and systems thinking (the ability to take one big idea and break it down into its component pieces).

Lesson 2: Webpages, Documents and Structure

HTML, CSS and Java Script Files

HTML files control the structure of a webpage, while CSS files are used to add style to a webpage (control how it looks) and Java Script files represent interactive components of the page.

HTML's "Tree Like" Structure

Typically programmers talk about the Document Object Model or "DOM" to refer to the tree like structure of a webpage. In general terms a tree structure or tree diagram is just a way of representing a structure in graphical form through layers or stages (branches).

Each HTMl tag creates an element in the DOM that the browser uses to display the page. An element is created from the starting tag to the ending tag and everything in between is content. The content can be empty, it can be text or it can be another element. Each tag has a name or type that defines what kind of element will be created. Each tag can also have attributes with values. To tie structure and style together the HTML document can provide the browser with information on what style to use and where to get it.

Text Editors

When writing code, programmers use special text editors. Some of them are Codepen, Scratchpad and Sublime. Some text editors give you automatic feedback while others need to be refreshed, others will automatically generate a closing HTML tag when you write an opening tag, some can handle just two or three languages-Scratchpad handles HTML and CSS while Codepen can handle HTML, CSS and Java Script-while some others like Sublime are more powerful and can understand any language.

Lesson 3: Adding CSS Style Structure

Understanding CSS

Cascading Style Sheets or "CSS" files/sheets are code written to control the style of HTML elements. One can have several CSS files or sheets that will be referenced in HTML. With CSS the most specific rule is applied to every element (Cascading).

In CSS you first write what is called a selector, which defines to what elements of the page, a particular style will apply. Another way to do so is to select elements by class. If you want to apply style to a class called "description"you do so by typing ".description {}" and inside the brackets you write the style in form of an attribute calling value. You can also combine slectors, so for example, you could select all links that are inside an element with the class "article".

Inheritance and Avoiding Repetition

Inheritances is the mechanism by which properties are applied not only to a specified element but also its descendants. This concept relies on the document tree, which is the hierarchy of HTML elements in a (web) page based on nesting. Descendent elements may inherit CSS property values from any ancestor alement enclosing them. In general, descendant elements inherit text-related properties, but box-related properties are not inherited. Inheritance allows certain properties from being declared over and over again in a tyle sheet, allowing software developers to write less CSS. It also enhances faster-loading of web pages while navigating.

In general, inheritance and avoiding repetition could be thought as two sides of the same coin. In software engineering and programming language theory, the abstraction principle (or the principle of abstraction) is a basic dictum that aims to reduce duplication of information in a program (usually with emphasis on code duplication) whenever practical by making use of abstractions provided by the programming language or software libraries. Source:[Wikipedia]. Avoiding repetition reduces computation time, and also enhances faster-loading of web pages while navigating.

Identation

In programming it refers to the nesting of related commands by creating a hierarchy similar to an outline used in English compositions classes. It is just a way of grouping commands together that form a subroutine, function, etc.

How to include "CSS Styling" in a Web Page

There are three methods to do this: a) Method 1: Write CSS in the head of yor HTML. To do this, at the top of your HTML document, add a "head" element, then, inside the "head" element add a "style" element. Write your CSS in the style element and then write the HTML code for the rest of your page below; b) Method 2: Write all your structural HTML in one file-let's call it "main.html"; In a separate file, let's call it "main.css" put all your CSS code; Add a "head" element to the top of your HTML and then add a "link" tag inside the "head" element, and finally, add the following attributes to your link [link rel="stylesheet" * type="text/css" * href="main.css"]; Method 3: Write your style inline with your HTML. With this method you modify the style attributes of every individual HTML element. (This is generally a bad idea because it leads to a lot of repetead code).

The Box Model

HTML elements are boxes and each box has four components: The content in the most inner part of the box, the padding that clears an area around the content and it is affected by the background color of the box-one can think of it as the space inside the box that protects the content directly, the border that goes around the padding and the content, and the outer layer, the margin and this clears an area around the border. The margin doesn't have a background color and is completely transparent. Because there are many components to each box, it can often be hard to get the size of each box right. But there are two techniques one can use to deal with sizing: a) Set sizes in terms of percentages rather than pixels; b) Set the box-sizing to "border-box"for every element.

Regarding box positioning we must take into account that "Div's" are block ekements-as opposed to inline elements; so by default they take up the entire width of the page. By adding the rule "display:lex" to the appropriate CSS will override this behaviour and let div's appear next to each other.
